# rishi.my Agent Brief

Read this before changing the repo.

This site is a minimal personal website: a quiet, static notebook for thoughts,
life notes, and things built. It should stay compact, text-first, and durable.
Do not turn it into a dashboard, portfolio template, social feed, or CMS app.

## Current Setup

- Static site generated by Jekyll and published through GitHub Pages.
- Content lives in Markdown/YAML files in Git.
- Editing is handled by Pages CMS through `.pages.yml`.
- `/admin` redirects to `https://app.pagescms.org/`.
- There is no custom backend, Cloudflare Worker, D1 draft store, GitHub-login
  editor, or in-site editing overlay.

## Site Shape

- `me`: homepage with intro, optional image, and contact links.
- `thought`: ideas, questions, essays, research notes.
- `life`: people, places, seasons, stories.
- `matter`: artifacts, projects, things built.

Keep the homepage minimal. Do not add section promos, hero sections, cards, or
marketing copy.

## Visual Language

- Geist Mono everywhere.
- Soft off-white background, graphite text, muted metadata, Source Blue accent.
- Left rail navigation on desktop, compact top navigation on mobile.
- Page headers use a small question heading plus a blue command subtitle.
- Archive pages use compact date plus title rows only.
- Detail pages use narrow article layouts with date metadata.
- Images should feel like evidence attached to a memo, not decoration.

Avoid logos, big names, cards, hero sections, gradients, decorative blobs,
heavy separators, oversized type, and one-off visual systems.

## Editing Rules

- Keep editing simple: use Pages CMS directly for content changes.
- `.pages.yml` is the source of truth for CMS fields.
- Media should go through the Pages CMS `images` media library and output under
  `/media`.
- `thought` and `life` filenames use `{year}-{month}-{day}-{primary}.md`.
- `matter` filenames use `{primary}.md`.
- Entry bodies should remain Markdown via the Pages CMS rich-text field.

Do not add a custom GitHub-login admin UI, client-side editor, Cloudflare
Worker, D1 drafts, local proxy, or new backend unless Rishi explicitly asks to
reopen that direction.

## Change Rubric

Before adding anything, ask:

- Does this make the site more durable?
- Does it reinforce the quiet personal notebook feel?
- Does it reduce noise or sharpen attention?
- Can it stay static?
- Will it still feel appropriate in ten years?

If a change makes the site feel like a generic blog, startup page, dashboard, or
social product, do not make it without explicit direction.

## Checks

- Preserve the `<!-- rishi.my: static, handwritten, no analytics -->` posture.
- Keep generated output, local toolchains, and private scratch files out of Git.
- Run `git diff --check` before finishing.
- Run `bundle exec jekyll build` when Ruby/Bundler is available.
